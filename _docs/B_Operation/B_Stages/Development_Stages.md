# Development Stages
This document defines the ordered stages of the project lifecycle.
Stages provide scope boundaries, objectives, and exit criteria.

## Stage Principles
- Stages are sequential and non-overlapping.
- Only one stage may be active at a time.
- Entering or exiting a stage requires explicit approval.
- Any change to stages must be logged.

## Stage Definition Template
Each stage MUST follow the structure below.

### Stage Name
A short, descriptive name.

### Objective
- The primary goal of this stage.
- The scope boundary of the stage.
- What success means for this stage.

### Scope
- What is explicitly included.
- What is explicitly excluded.
- What must not be addressed in this stage.

### Entry Criteria
- Conditions that must be met to enter this stage.
- Required approvals or decisions.
- Required documentation state.

### Exit Criteria
- Conditions that must be met to exit this stage.
- Required deliverables.
- Required validations.
