# Stage Exit Checklists
This document defines the mandatory exit checks for each development stage.
A stage may not be exited unless all its checklist items are satisfied.

## Exit Principles
- Exit criteria are objective and verifiable.
- A stage cannot be partially exited.
- Exit approval is required before transitioning to the next stage.
- Failing any item blocks stage transition.

## Checklist Template
Each stage MUST have a checklist using the structure below.

### Stage Name
The name of the stage this checklist applies to.

### Exit Checklist
- All objectives of the stage are fulfilled.
- All required documentation is complete and approved.
- All blocking decisions are logged.
- All critical tasks are completed or explicitly deferred.
- No unresolved contradictions remain.

### Verification
- Evidence or references proving checklist completion.
- Reviewer or approver identity.
- Approval date.
