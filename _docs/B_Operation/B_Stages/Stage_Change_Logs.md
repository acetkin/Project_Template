# Stage Change Logs
This document records all approved changes related to project stages.
It is append-only and must never be rewritten or reordered.

## Logging Rules
- Only approved stage changes may be logged.
- Each change must be logged immediately after approval.
- Entries must not be edited or removed once written.
- Each entry must reference the affected stage(s).

## Stage Change Entry Format
Each entry MUST follow the structure below.

### Change ID
A unique incremental identifier.

### Date
YYYY-MM-DD.

### Change Type
One of: Stage Added, Stage Modified, Stage Removed, Stage Transition.

### Description
A concise description of what changed.

### Rationale
Why the change was necessary.

### Impact
What this change affects in scope, sequencing, or deliverables.

### Affected Stages
List of stage names impacted by this change.
