# Release Checklist
This document defines the mandatory checks required before a stage completion or release.
A release may not proceed unless all checklist items are satisfied and approved.

## Release Principles
- A release represents a stable and reviewable state.
- All checklist items must be verifiable.
- Partial releases are not allowed unless explicitly approved.
- Failing any item blocks the release.

## Checklist Template
Each release MUST use the structure below.

### Release Identifier
A unique identifier for the release or stage completion.

### Scope
- What this release includes.
- What this release explicitly excludes.
- Any deferred items carried forward.

### Documentation Checks
- All required documentation is complete.
- All documentation changes are approved.
- Context_Packet.md reflects the final approved state.

### Decision Checks
- All scope- or direction-affecting decisions are logged.
- No pending unlogged decisions exist.
- Decision log references are complete.

### Task Checks
- All tasks in scope are completed or explicitly deferred.
- Task_Board.md accurately reflects task status.
- No critical tasks remain blocked.

### Validation Checks
- Required reviews are completed.
- Required validations or tests are completed.
- Known risks are documented and accepted.

### Approval
- Approver identity.
- Approval date.
- Approval notes.
